# Agent Maestro â€” VPS Deployment

One-command deploy + data migration for running Agent Maestro in a browser on a VPS.

## Prerequisites

**Local machine:**
- `bash`, `rsync`, `ssh`, `openssl`
- Access to the `feat/web-ui-deploy` branch of this repo

**VPS:**
- Ubuntu 20.04+ (or any Debian-based distro)
- Root or sudo access (deploy.sh uses `ssh` as the given user; some steps need sudo â€” run as a user with passwordless sudo or as root)
- Inbound ports: **22** (SSH), **80** (HTTP), **443** (HTTPS for TLS)

---

## Two-command quickstart

```bash
# 1. Deploy everything (provision + build + service + nginx)
./deploy/deploy.sh ubuntu@1.2.3.4 -i ~/.ssh/my_key --password 'MySecretPass' --domain maestro.example.com

# 2. Migrate all local maestro data to the VPS
./deploy/migrate-data.sh ubuntu@1.2.3.4 -i ~/.ssh/my_key
```

After these two commands, open `http://your-domain/` in a browser.

## deploy.sh

```
Usage: ./deploy/deploy.sh <user@host> [OPTIONS]

Options:
  -i <ssh_key>          SSH private key path
  --password <pass>     App password (MAESTRO_AUTH_PASSWORD). Prompted if omitted.
  --domain <fqdn>       FQDN for nginx server_name (default: catch-all _)
  --port <n>            App port for maestro-server (default: 4570)
  --repo-url <url>      Git remote to clone from (default: rsync local working tree)
```

**What it does (idempotent, fail-fast):**

1. **Provision** â€” installs Node LTS, git, build tools, rsync, nginx, Claude CLI
2. **Sync** â€” rsyncs the repo (or `git clone` if `--repo-url` is given)
3. **Build** â€” `bun install`, then builds `maestro-server` (tsc), `maestro-ui` (`build:web`), `maestro-cli` (tsc)
4. **Env file** â€” writes `/etc/maestro/maestro.env` (chmod 600) with all runtime + auth vars; generates `MAESTRO_AUTH_SECRET` via `openssl rand -hex 32`
5. **Systemd** â€” installs and starts `maestro-server.service` running under **node** (not bun)
6. **Nginx** â€” configures a reverse proxy with WebSocket upgrade headers for `/ws` and `/pty`

> **Runtime is node, not bun.** `node-pty`'s `onData` does not fire under bun, and bun strips the spawn-helper exec bit. Build steps use bun; the service runs `node maestro-server/dist/server.js`.

## migrate-data.sh

```
Usage: ./deploy/migrate-data.sh <user@host> [OPTIONS]

Options:
  -i <ssh_key>           SSH private key path
  --from <dir>           Local maestro dir (default: ~/.maestro)
  --to   <dir>           Remote maestro dir (default: /home/<user>/.maestro)
  --mirror               Also DELETE remote files not in source (use with care)
```

**What it syncs:**
- `~/.maestro/data/` â€” projects, tasks, task-lists, team-members, teams, spells, orderings, indexes
- `~/.maestro/sessions/` â€” all session logs and manifests

The service is stopped before sync and restarted after. Safe to re-run.

## Enabling TLS (strongly recommended)

Auth over plain HTTP sends the password in cleartext. Enable HTTPS:

```bash
# On the VPS
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d maestro.example.com
```

certbot rewrites the nginx config to add a `:443` listener and HTTPâ†’HTTPS redirect. Auto-renewal is usually pre-configured via `certbot.timer`.

## After deployment: run claude login

The Claude CLI is installed but **not authenticated**. SSH into the VPS and run:

```bash
ssh ubuntu@1.2.3.4
claude login
```

This must be done manually â€” the deploy script intentionally does not run it.

## Troubleshooting

```bash
# Service status
systemctl status maestro-server

# Live logs
journalctl -u maestro-server -f

# Test server directly (bypassing nginx)
curl http://localhost:4570/api/projects

# Nginx logs
tail -f /var/log/nginx/error.log

# Rebuild after code change (on the VPS)
cd ~/maestro
bun install
cd maestro-server && bun run build && cd ..
cd maestro-ui && bun run build:web && cd ..
cd maestro-cli && bun run build && cd ..
systemctl restart maestro-server
```
